# Spec-Driven Development

## Терминология

### Memory Bank
**Memory Bank (банк памяти)** — это постоянное, структурированное и общее долговременное хранилище знаний о проекте. В контексте спецификационно-ориентированной разработки с участием ИИ он решает главную проблему искусственного интеллекта — «амнезию» между сессиями. Благодаря Memory Bank каждый новый диалог или задача начинаются не с чистого листа, а с полным пониманием контекста проекта.

У Claude Code есть **две встроенные системы памяти**, работающие по принципу «инструкции, которые вы пишете» и «заметки, которые ИИ пишет сам»:

| Система | Кто пишет | Содержание | Где хранится |
|---------|-----------|------------|--------------|
| **CLAUDE.md** | Вы (разработчик) | Инструкции, стандарты кодинга, архитектура, команды сборки | Файлы в проекте (`./CLAUDE.md`) или глобально (`~/.claude/CLAUDE.md`) |
| **Auto Memory** | Claude (автоматически) | Выученные паттерны, отладочные инсайты, предпочтения по стилю | Файлы в `.claude/projects/` |

CLAUDE.md загружается в контекст при каждом запуске сессии, а Auto Memory позволяет Claude сохранять информацию, которую он считает полезной для будущих сессий, без вашего вмешательства.

---

### 🧠 Сторонние решения: расширенный банк памяти

Помимо встроенных механизмов, существует множество MCP-серверов (Model Context Protocol), которые добавляют Claude Code **постоянную память с расширенными возможностями**:

*   **am-memory**: SQLite-база с гибридным поиском (BM25 + векторный). Автоматически инжектирует релевантные воспоминания в каждый промпт через хуки. Установка одной командой: `pip install am-memory && am init`.
*   **claude-memory-fts**: Аналогично, но с акцентом на семантический поиск через локальные эмбеддинги (all-MiniLM-L6-v2). Ранжирует факты по частоте доступа, свежести и категории.
*   **memory-bank-mcp**: Интеграция через MCP с поддержкой графа знаний — Claude может искать связи между сущностями проекта, например: «найди в графе знаний всё, что связано с аутентификацией».
*   **claude-memory (npm)**: Легковесное решение с типами памяти (rules, architecture, decisions) и возможностью экспорта/импорта в JSON.
*   **russbeye/claude-memory-bank**: Готовая структура с папками для решений (ADR), паттернов, архитектуры и известных проблем, а также специализированными агентами и командами типа `/update-memory-bank`.
*   **hindsight-cc**: Плагин для Claude Code на основе векторной БД Hindsight. Автоматически сохраняет промпты и транскрипты сессий, а при старте новой сессии инжектирует релевантный контекст из прошлого.

> **Важно**: Большинство этих решений работают **локально** (данные не покидают ваш компьютер) и используют **MCP**, который Claude Code поддерживает из коробки.

---

### ⚙️ Как это работает на практике (пример)

1.  **Установка MCP-сервера** (например, через `claude mcp add memory -- npx claude-memory-fts`).
2.  **Сохранение факта**: вы говорите Claude: «Запомни, что мы перешли с Express на Hono в июне 2026».
3.  **Загрузка контекста**: при следующей сессии Claude автоматически или по вашей команде (`/catch-up` или `Load my memory for the project`) подгружает сохранённые факты.
4.  **Автоматическая запись**: некоторые системы через хуки сами сохраняют важные решения в процессе работы.

---

### 💡 Когда что использовать

*   **Для быстрого старта** — встроенные `CLAUDE.md` и `auto memory`.
*   **Для сложных проектов** — MCP-серверы с SQLite и поиском (`am-memory`, `claude-memory-fts`).
*   **Для структурированного хранения архитектурных решений (ADR)** — `claude-memory-bank` от russbeye.
*   **Для максимальной автоматизации** — `hindsight-cc` с фоновым сохранением всех сессий.

Таким образом, Memory Bank для Claude Code — это не единая вещь, а экосистема из встроенных и сторонних решений, которые превращают Claude из статичного ассистента в партнёра, помнящего историю вашего проекта.
