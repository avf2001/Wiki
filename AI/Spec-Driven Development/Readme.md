# Spec-Driven Development

## Терминология

**Memory Bank (банк памяти)** — это постоянное, структурированное и общее долговременное хранилище знаний о проекте. В контексте спецификационно-ориентированной разработки с участием ИИ он решает главную проблему искусственного интеллекта — «амнезию» между сессиями. Благодаря Memory Bank каждый новый диалог или задача начинаются не с чистого листа, а с полным пониманием контекста проекта.

У `Claude Code` есть две встроенные системы памяти, работающие по принципу «инструкции, которые вы пишете» и «заметки, которые ИИ пишет сам»:

| Система	| Кто пишет |	Содержание | Где хранится |
|-|-|-|-|
| CLAUDE.md	| Вы (разработчик) | Инструкции, стандарты кодинга, архитектура, команды сборки |	Файлы в проекте (./CLAUDE.md) или глобально (~/.claude/CLAUDE.md) |
| Auto Memory	| Claude (автоматически) | Выученные паттерны, отладочные инсайты, предпочтения по стилю | Файлы в .claude/projects/ |

`CLAUDE.md` загружается в контекст при каждом запуске сессии, а `Auto Memory` позволяет `Claude` сохранять информацию, которую он считает полезной для будущих сессий, без вашего вмешательства.
